## Mission Control Image Upload Fix (Base64 Issue)

**Problem:** When uploading images to core-cards and image widgets, the base64 data string was being stored directly as the image "name", creating strings with 50,000+ characters starting with `Card Image data:image/png;base64,iVBORw0KGgoAAAANSUhEUgA...`

**Root Cause:** The BlockSettingsPanel was using plain `<Input>` fields for image URLs instead of the `ImageUpload` component that properly handles file uploads.

**Solution:** 
- Modified `client/src/components/page-builder/BlockSettingsPanel.tsx`
- Added import for ImageUpload component
- Replaced Input fields with ImageUpload for:
  - `case 'image'`: Image block URL field
  - `case 'cards'`: Each card's image field
- ImageUpload component already handles proper file upload to `/api/admin/uploads` endpoint
- Returns clean URLs like `/uploads/filename.jpg` instead of base64 data
