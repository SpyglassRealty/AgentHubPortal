## Technical Patterns Learned

### QA Workflow
1. **Audit first** - Always read relevant files and post findings
2. **Draft before commit** - Show exact changes for approval  
3. **Wait for go signal** - Never commit without explicit ✅
4. **Verify deployment** - Confirm READY state via vercel list
5. **Report status** - SHA + deployment ID + READY confirmation

### Spyglass IDX Architecture
- **Hosting:** Vercel (separate from Mission Control on Render)
- **CMS:** Mission Control API for dynamic content
- **Blog system:** Dual endpoints (blog-index + individual slugs)
- **Navigation:** Header.tsx with defaultNavigation fallback
- **Phone number:** 737-727-4889 is correct main number

### Code Quality
- Always use AbortController for timeouts (not just Next.js revalidate)
- Add pagination support defensively (check multiple metadata fields)
- Replace silent catch blocks with detailed error logging
- Verify deployments actually reach READY state before reporting success
