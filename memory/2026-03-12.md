## Technical Patterns Learned

### Prop Chain Debugging
When React components silently fail:
1. Verify prop is declared in interface
2. Check prop is destructured in component signature  
3. Confirm prop is passed through all intermediate components
4. Use conditional checks (`if (onCallback && ...)`) to prevent silent failures

### Drizzle Schema Sync Issues
- Database columns can exist but be missing from Drizzle schema
- Missing schema columns cause `undefined` references and SQL syntax errors
- Always sync schema after running migrations

### Force-Assign vs Regular Assign Patterns
- Force-assign: `userId: null`, uses `assignedName`/`assignedEmail` columns
- Regular assign: `userId: <valid_id>`, joins with users table
- Queries must use `leftJoin` to include both patterns
- Frontend display logic: `s.userId ? s.firstName : s.assignedName?.split(' ')[0]`
