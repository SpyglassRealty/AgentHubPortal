## Spyglass IDX - Infinite Fetch Loop Fix
- **Critical bug**: `useListings` hook had an infinite API call loop — 167 fetch calls in 10 seconds
- **Root cause**: `useCallback` for `fetchListings` depended on `currentFilters` state. Every fetch completion updated `currentFilters`, which recreated the callback, which triggered the parent's `useEffect`, which called `fetchListings` again
- **Symptoms**: Listings flickered between loading skeleton and actual content, photos appeared to not load, massive API bill risk
- **Fix**: Rewrote `useListings` hook to use `useRef` for internal state (`currentFilters`, `page`, `loading`, `hasMore`) so the callback has a stable identity with zero dependencies. Added concurrent fetch guard.
- **Result**: Zero additional API calls after initial page load (verified with 10-second monitoring)
- Commits: multiple, final deploy at spyglass-idx.vercel.app
